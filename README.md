# What's in My Fridge?

A scalable Flutter mobile app for food inventory, expiration tracking, AI-assisted fridge recognition, recipe generation, shopping lists, meal planning, statistics, and conversational cooking help.

## Included

- Material Design 3 with light, dark, and system themes
- Email/password authentication, password reset, Google Sign-In, and Apple Sign-In
- Demo mode when Firebase has not been configured
- Dashboard with inventory and expiration KPIs
- Inventory CRUD, category filtering, search, barcode scanning, food images, and AI photo recognition
- AI recipe generation that prioritizes food expiring soon
- Dietary filters: vegetarian, vegan, gluten-free, high protein, low carb, under 30 minutes
- Expiration timeline and FCM notification backend for today, tomorrow, and three days before
- Shopping list from missing ingredients, repeated purchases, and low stock
- Drag-and-drop weekly meal planner
- Statistics and monthly reporting UI
- Natural-language AI assistant
- Firestore and Storage security rules, indexes, Emulator Suite configuration, and TypeScript Cloud Functions

## Architecture

```text
lib/
├── core/                 # Theme, shared widgets, formatters
├── domain/               # Entities and repository contracts
├── data/                 # Firebase/demo repositories and services
└── presentation/         # Riverpod state, screens, reusable UI
functions/
└── src/index.ts          # AI callables and scheduled notifications
```

The presentation layer never talks directly to Firestore or AI endpoints. Riverpod providers expose controllers, domain repositories define contracts, and the data layer supplies Firebase or local demo implementations.

## Run the app

This package contains the complete application source. Generate Flutter's native platform boilerplate once:

```bash
flutter create . --platforms=android,ios
flutter pub get
flutter run
```

Without Firebase configuration, any valid-looking email and an 8-character password open the local demo mode. This is intentional so the UI and product flows can be reviewed immediately.

## Connect Firebase

Install the Firebase CLI and FlutterFire CLI, then configure the project:

```bash
firebase login
dart pub global activate flutterfire_cli
flutterfire configure
firebase init firestore storage functions emulators
```

Enable these Firebase Authentication providers:

- Email/Password
- Google
- Apple

For Android Google login, add the app's SHA-1 and SHA-256 fingerprints in Firebase. For iOS, enable the **Sign in with Apple** capability and configure the URL schemes generated by Firebase/Google.

Deploy database and storage configuration:

```bash
firebase deploy --only firestore:rules,firestore:indexes,storage
```

## AI backend

The callable Cloud Functions use Gemini through a server-side secret. No AI key is stored in the mobile application.

```bash
firebase functions:secrets:set GEMINI_API_KEY
cd functions
npm install
npm run build
cd ..
firebase deploy --only functions
```

The callables have App Check enforcement enabled. Register Android and Apple App Check providers before production rollout. During local development, either use App Check debug tokens or temporarily set `enforceAppCheck` to `false` in `functions/src/index.ts`.

## Push notifications

On first authenticated launch, the app requests notification permission and stores the FCM token under `users/{uid}`. The scheduled function runs daily at 08:00 Europe/Berlin and checks foods expiring:

- today
- tomorrow
- in three days

For Android, create a notification channel named `expiration_alerts`. For Apple platforms, upload an APNs authentication key in Firebase and enable Push Notifications and Background Modes in Xcode.

## Firestore layout

```text
users/{uid}
  fcmTokens: string[]
  notificationPreferences: { today, tomorrow, threeDays }

users/{uid}/inventory/{itemId}
  name, category, quantity, unit
  purchaseDate, expirationDate
  storageLocation, imageUrl, notes, createdAt
```

Additional production collections can follow the same user-scoped pattern:

```text
users/{uid}/recipes/{recipeId}
users/{uid}/shopping/{itemId}
users/{uid}/mealPlan/{entryId}
users/{uid}/reports/{monthId}
```

## Production checklist

- Replace the demo repository with Firestore-only behavior for release builds if offline demo access is not desired.
- Enable Firebase App Check, Crashlytics, Performance Monitoring, and Analytics consent handling.
- Add localized strings through Flutter `gen_l10n`.
- Add Open Food Facts or another licensed product database behind a server-side barcode lookup service.
- Add integration tests for authentication, image upload, AI confirmation, and notification deep links.
- Review AI output, allergen handling, food-safety wording, privacy disclosures, and data-retention policy before store submission.
